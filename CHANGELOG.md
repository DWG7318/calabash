# Changelog

## 2.2.0

- Make Calabash BI mandatory in the project-initiation transaction, before material
  source preparation or Owner questioning.
- Add a compact, persistent Calabash Definition Window and a full audit dashboard.
- Require continuous visibility of the current accepted definition, latest semantic
  changes, working-draft state, immediate attention items, and freshness identity.
- Keep accepted baseline, provisional/experimental decisions, proposed amendments,
  observed UI/Workflow drift, superseded history, and Git working drafts visually
  distinct.
- Add `CALABASH_BI_BOOTSTRAP_PASS`, `CALABASH_BI_LIVE`,
  `BI_VISIBILITY_DEGRADED`, stale/error states, and project-open visibility gates.
- Add project-local bootstrap, build, desktop runtime, semantic change-recording,
  doctor, local read-only server, launcher, and shortcut tools.
- Add native compact-window, compact-browser, and declared headless publication modes.
- Add state hashing, automatic bounded refresh, runtime health, Git draft visibility,
  and reproducible BI snapshots.
- Make BI freshness a gate for Owner decisions, baseline freeze, Loop activation,
  review, amendment adoption, and final closure.

## 2.1.0

- Add provisional/prototype decisions, UI Reality and Core Workflow Reality living
  tracks, append-only lineage, periodic review, and Loop integration.
